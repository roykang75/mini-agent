세 가지 방식 한눈에 비교
================================================================================

1️⃣ gather()                    2️⃣ gather(return_exceptions=True)    3️⃣ TaskGroup()
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━
(사용 금지 ⚠️)               (권장 👍)                         (새 표준 👍)

호출법:                        호출법:                          호출법:
await asyncio.gather(...)      await asyncio.gather(...,        async with asyncio.TaskGroup() as tg:
                               return_exceptions=True)             tg.create_task(...)

예외 발생 시:                 예외 발생 시:                    예외 발생 시:
❌ 즉시 raise                ✅ raise 안 함                    ✅ ExceptionGroup raise
❌ 나머지는 백그라운드       ✅ 모두 끝까지 실행             ✅ 나머지는 취소됨

결과:                         결과:                           결과:
❌ 폐기됨                     ✅ [결과1, Exception, 결과3]    ⚠️  취소되어 조회 불가

타이밍:                       타이밍:                         타이밍:
⚡ 0.1초 (예외 시점)         ⏱️  1.0초 (최장 작업)           ⚡ 0.1초 (예외 시점)

리소스 누수:                  리소스 누수:                    리소스 누수:
💥 있음                       ✅ 없음                         ✅ 없음

용도:                         용도:                           용도:
⚠️  거의 없음                 📡 웹 요청, API 호출            🔐 DB, 배포, 트랜잭션
                              🌐 크롤링                        🔒 원자적 작업
                              📦 배치 처리

Python 버전:                  Python 버전:                    Python 버전:

실행 결과 비교 (같은 작업, 다른 방식)
================================================================================

Task: 0.5초 작업, 0.1초 후 실패, 1.0초 작업

┌──────────────────────────────────────────────────────────────────────┐
│ 1️⃣  gather()                                                       │
├──────────────────────────────────────────────────────────────────────┤
│ t=0.0초: task1(0.5s), fail(0.1s), task3(1.0s) 시작                 │
│ t=0.1초: ❌ ValueError 즉시 발생                                    │
│          → 호출자: 0.1초에 예외 처리                               │
│          → task1: 백그라운드에서 여전히 0.5초 대기 중 (누수!)      │
│          → task3: 백그라운드에서 여전히 1.0초 대기 중 (누수!)      │
│ t=0.5초: task1 백그라운드 완료 (결과는 버려짐)                    │
│ t=1.0초: task3 백그라운드 완료 (결과는 버려짐)                    │
│                                                                      │
│ 결과: ValueError() 만 반환                                         │
│ 문제: ⚠️ 리소스 누수, 결과 손실, 예측 불가능                      │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ 2️⃣  gather(return_exceptions=True)                                │
├──────────────────────────────────────────────────────────────────────┤
│ t=0.0초: task1(0.5s), fail(0.1s), task3(1.0s) 시작                 │
│ t=0.1초: fail() 예외 발생 (raise하지 않음)                        │
│ t=0.5초: task1 완료                                                │
│ t=1.0초: task3 완료                                                │
│          → 호출자: 1.0초에 결과 수신                               │
│                                                                      │
│ 결과: [task1_결과, ValueError(...), task3_결과]                   │
│ 장점: ✅ 모든 작업 완료, 결과 수집, 명확한 상태                   │
│ 리소스: ✅ 안전함                                                  │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ 3️⃣  TaskGroup()                                                    │
├──────────────────────────────────────────────────────────────────────┤
│ t=0.0초: task1(0.5s), fail(0.1s), task3(1.0s) 시작                 │
│ t=0.1초: fail() 예외 발생 → task1, task3에 CancelledError          │
│ t=0.1초: task1, task3 취소됨                                        │
│          → 호출자: 0.1초에 ExceptionGroup raise                     │
│                                                                      │
│ 결과: ExceptionGroup([ValueError(...)])                           │
│ 용도: DB 트랜잭션, 배포 (원자적 실행 필요)                        │
│ 리소스: ✅ 안전함                                                  │
└──────────────────────────────────────────────────────────────────────┘

패턴 1: "한 개 실패해도 나머지는 끝까지" (웹 스크래핑, API 호출)
────────────────────────────────────────────────────────────────

    results = await asyncio.gather(
        fetch_url("url1"),
        fetch_url("url2"),
        fetch_url("url3"),
        return_exceptions=True  # ← 핵심!
    )
    
    successful = []
    failed = []
    
    for result in results:
        if isinstance(result, Exception):
            failed.append(result)
        else:
            successful.append(result)
    
    print(f"성공: {len(successful)}, 실패: {len(failed)}")


패턴 2: "하나 실패 시 전부 취소" (DB, 배포)
──────────────────────────────────────────

    try:
        async with asyncio.TaskGroup() as tg:
            tg.create_task(update_db1())
            tg.create_task(update_db2())
            tg.create_task(update_db3())
    except* Exception as eg:
        print(f"실패: {eg.exceptions}")
        # 모든 업데이트가 자동 취소됨 (일관성 보장)


패턴 3: 타임아웃과 함께 사용
──────────────────────────

    try:
        results = await asyncio.wait_for(
            asyncio.gather(*tasks, return_exceptions=True),
            timeout=5.0
        )
    except asyncio.TimeoutError:
        print("작업이 5초 내 완료되지 않음")

================================================================================
주의사항 체크리스트
================================================================================

❌ 하면 안 되는 것:
   [ ] gather()를 명시적 옵션 없이 사용 (리소스 누수)
   [ ] 결과를 받을 계획 없이 gather() 사용
   [ ] return_exceptions=True 없이 예외 처리 기대
   [ ] TaskGroup에서 CancelledError를 놓치기

✅ 해야 할 것:
   [✓] return_exceptions=True 또는 TaskGroup 중 하나 선택
   [✓] 독립적 vs 원자적 작업 구분하기
   [✓] 예외/성공 결과 명확히 분류하기
   [✓] 타임아웃 고려하기
   [✓] 부분 성공 시나리오 테스트하기

웹 요청 여러 개 (권장 패턴):
──────────────────────────

    async def fetch_multiple_urls(urls: list[str]) -> dict:
        results = await asyncio.gather(
            *[fetch(url) for url in urls],
            return_exceptions=True
        )
        
        data = {}
        errors = []
        
        for url, result in zip(urls, results):
            if isinstance(result, Exception):
                errors.append((url, str(result)))
            else:
                data[url] = result
        
        return {"data": data, "errors": errors}


DB 배치 업데이트 (TaskGroup):
──────────────────────────────

    async def batch_update_users(updates: dict) -> bool:
        try:
            async with asyncio.TaskGroup() as tg:
                for user_id, data in updates.items():
                    tg.create_task(update_user(user_id, data))
            
            logger.info("모든 사용자 업데이트 완료")
            return True
        
        except ExceptionGroup as eg:
            logger.error(f"업데이트 실패: {eg.exceptions}")
            # 트랜잭션 자동 롤백
            return False
