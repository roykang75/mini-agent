## Design Context

### Users
- 개발자 본인 (1인 사용)
- Agent에게 자연어로 명령을 내리고, 툴 호출 과정과 결과를 실시간으로 확인하는 컨텍스트
- 향후 Human-in-the-Loop(HIL)에 따라 승인/거부 등 인터랙티브 컴포넌트가 추가될 수 있음

### Brand Personality
- Technical, Functional, Clean
- 터미널처럼 기술적이면서도 밝고 읽기 편한 인터페이스
- 레퍼런스: Claude Code VSCode Plugin

### Aesthetic Direction
- 라이트 모드 전용
- 밝고 경쾌한 톤 (dark/moody 아님)
- 터미널 감성: 모노스페이스 폰트 활용, 툴 호출/결과는 코드 블록 스타일
- 채팅 메시지는 가독성 우선의 산세리프 폰트
- Anti-reference: 없음

### Tech Stack
- Next.js (App Router)
- Tailwind CSS
- SSE 스트리밍 (fetch + ReadableStream)

### Design Libraries
- shadcn/ui — 컴포넌트 기반 UI
- Lucide Icons — 아이콘
- CVA (Class Variance Authority) — 컴포넌트 variants 관리
- tw-animate-css — Tailwind 애니메이션

### Design Principles
1. **Content First** — 장식보다 메시지와 툴 출력의 가독성을 최우선으로 한다
2. **Terminal Aesthetic** — 툴 호출/결과는 터미널 느낌의 모노스페이스 블록으로 표현한다
3. **Progressive Disclosure** — thinking, tool_call, tool_result 등 중간 과정은 접거나 축소할 수 있게 한다
4. **Component Ready** — HIL 컴포넌트(승인 버튼, 파일 미리보기 등)가 자연스럽게 끼워 넣어질 수 있는 구조를 유지한다
5. **Light & Bright** — 밝은 배경에 충분한 대비, 시각적 피로를 줄이는 여백 활용
